# Upgrade

:::info

This document describes the supported version compatibility and upgrade paths for Alauda Container Platform Logging Service.

:::

## Compatibility Matrix

The table below lists supported versions of the Alauda Container Platform Logging Service:

| Logging Service Version | Supported Alauda Container Platform Version |
| --- | --- |
| 4.3.0 | 4.3.0 |
| 4.2.x | 4.2.x |

:::info
Logging 4.3.0 is compatible only with ACP 4.3.0. Planned support for ACP 4.2.5 starts in Logging 4.3.1.
:::

## Upgrade Path Guidelines

### Upgrade Alongside ACP

- **Description**: In ACP 4.1.x and earlier releases, Logging Service plugins were lifecycle-aligned with ACP and were upgraded as part of the ACP release. For the following upgrade paths, the plugins are upgraded alongside ACP to version 4.3.0 during the ACP upgrade process.
- **Supported ACP Upgrade Paths**:
  - 4.0.x -> 4.3.0
  - 4.1.x -> 4.3.0
- **Result**: After the upgrade is completed, all Logging Service plugins are upgraded to version 4.3.0.

### Major Version Upgrade

- **Description**: Logging Service plugins support upgrades across minor versions after ACP is upgraded to a compatible version.
- **Prerequisite**: ACP must already be running on version 4.3.x.
- **Example**: `4.2.x -> 4.3.x`

### Patch-Level Upgrade
- **Description**: Upgrades between patch versions within the same minor version are fully compatible and can be performed directly.
- **Example**: `4.3.0 -> 4.3.x`
